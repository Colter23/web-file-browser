# Web File Browser 任务清单

这份清单把项目边界拆成可执行任务。后续开发应同时遵守
[PROJECT_BOUNDARY.md](PROJECT_BOUNDARY.md) 和 [ROADMAP.md](ROADMAP.md)。

项目定位保持不变：面向局域网的轻量单管理员文件管理器，优先保障 Linux Docker 部署、文件系统性能和行为可预测性。

## 基础约定

- 项目注释、文档、日志、错误提示、前端提示性文本和管理界面文案统一使用中文。
- 新增或修改功能时，触及到的旧英文注释、提示文案、日志和错误信息需要同步中文化。
- API 路径、环境变量、协议字段和代码标识符可以继续使用英文。
- 当前处于开发期，不为了兼容旧模型或旧接口增加复杂度。
- 发现模型、路由或数据结构不合理时，允许破坏式重构，并同步收敛前后端调用。
- 破坏式重构后应更新测试、README 和边界文档，不保留纯兼容包袱。

## 协作约定

- 本线程后续默认只推进后端：Rust 代码、服务层、路由层、模型、配置、部署脚本、接口文档和后端验证。
- 另一个 AI 负责前端页面、组件、交互和视觉实现；本线程不把前端 UI 改造作为默认任务。
- 后端接口发生破坏式调整时，本线程需要在文档中留下给前端协作者可直接对接的契约说明。
- 需要前端配合时，任务清单应写清楚接口路径、请求方法、请求字段、响应字段、错误码和默认行为。
- 除非用户明确要求或为了修通构建的最小网络层适配，本线程不主动修改 `ui/` 目录。

## 必须新增

1. 冲突策略。
   - 默认使用 `autoRename`。
   - 支持 `autoRename`、`reject` 和显式 `overwrite`。
   - 覆盖新建、上传、复制、移动、压缩输出、解压输出和回收站恢复。
   - `overwrite` 不能成为默认值，必须由配置或请求明确指定。
   - 文件覆盖替换应尽量先保留旧目标，新文件成功落位后再清理旧目标；失败时应保留旧目标和新源文件。

2. 在线编辑策略后续增强。
   - 可配置的最大可编辑文件大小已完成基础实现。
   - 文本/二进制安全判断已完成基础实现。
   - 默认不把二进制文件当作文本编辑。
   - 可配置可编辑扩展名或 MIME 分组已完成基础实现，默认不配置时保持自动文本判断。

3. 压缩和解压后台任务。
   - 新增 `POST /api/tasks/archive` 已完成基础实现。
   - 新增 `POST /api/tasks/extract` 已完成基础实现。
   - 已支持 `tar.gz`、`tgz` 和常用 `zip`。
   - 文件读写保持按块推进。
   - 压缩源路径和递归条目已拒绝符号链接和不支持的特殊文件。
   - 解压已防路径穿越、Zip Slip、过大解压、过多文件、过深路径和可疑条目。

4. Linux Docker 部署。
   - 前端和 Rust 后端多阶段构建 Dockerfile 已新增。
   - `.dockerignore` 已新增。
   - Docker Compose 示例已新增。
   - UID/GID、权限、只读挂载、数据卷、环境变量、备份和迁移文档已新增。
   - Docker 冒烟脚本 `scripts/docker-smoke.sh` 已新增。
   - `scripts/docker-smoke.sh` 已在真实 Linux Docker 环境通过，并据此修正前端构建、健康检查依赖、UID/GID 和跨挂载回收站恢复问题。
   - 性能冒烟脚本 `scripts/docker-perf-smoke.sh` 已新增，用于验证 1 万项目录、大文件流式传输、Range、压缩解压和回收站恢复。

5. 单管理员维护能力。
   - 登录后修改管理员密码 API 已完成基础实现。
   - 保持单管理员，不增加用户、角色或权限系统。
   - 服务端会话默认 7 天有效，并在登录、鉴权和指标统计时懒清理过期会话。
   - 默认 CORS 使用同源策略已完成基础实现。
   - 跨域只通过 `WEB_FILE_BROWSER_CORS_ORIGINS` 显式开启，不支持 `*`。
   - 默认不信任 `X-Forwarded-For`；可信反向代理部署必须显式开启 `WEB_FILE_BROWSER_TRUST_PROXY_HEADERS`。

## 必须完善

1. 目录浏览性能。
   - 默认保持 `detail=basic`。
   - 非必要时不读取 size 和 modified。
   - 仅在 `detail=full` 或按大小/修改时间排序时补充 stat。
   - 目录总数只在 `includeTotal=true` 时返回。
   - 元数据 JSON 响应复用同一次序列化结果生成 ETag 和响应体，`Last-Modified` 直接从元数据结构计算，避免大目录响应双重序列化和 JSON 反解析。
   - 元数据条件请求支持 `If-None-Match` 和 `If-Modified-Since`，命中时返回 `304`。
   - 默认目录响应复用目录自身 mtime 生成 `Last-Modified`，不为了响应头额外 `stat` 子项。
   - 没有 `If-None-Match` 且 `If-Modified-Since` 命中时，尽量在序列化目录 JSON 前返回 `304`。
   - `detail=basic` 和 `detail=full&sort=name` 的分页读取只保留当前页所需的排序窗口，仍完整扫描目录保证排序和 `hasMore` 正确，但减少大目录内存占用。
   - 目标是 1 万项目录稳定可用，更大目录尽量可用但不引入复杂架构。

2. 后台任务可观测性。
   - 记录当前处理路径、已处理字节、可低成本得到的总字节、速度、错误和部分成功。
   - 任务状态已增加 `currentPath`，复制、移动、删除、压缩和解压会在已有处理循环中更新当前处理条目，不额外预扫目录。
   - 在文件块或目录条目之间更及时检查取消状态。
   - 压缩、解压和复制已在目录条目、文件块、写入前、限速等待期间和收尾前补充取消检查。
   - 移动任务保持轻量文件系统移动，但进入真实移动前会重新检查取消状态。
   - 复制任务已改为先写入同目录临时路径，成功后再移动或替换目标；取消或失败会清理临时输出，覆盖复制不会破坏旧目标。
   - 删除任务不为了显示字节数预扫目录，避免删除大目录前产生额外磁盘遍历。
   - 任务状态保存在内存中，只保留最近任务历史。
   - 已增加 `WEB_FILE_BROWSER_TASK_HISTORY_LIMIT`，默认保留最近 200 条已结束任务，排队和运行中的任务不参与历史清理。
   - 已增加 `POST /api/tasks/cleanup`，可主动清理已结束任务历史，不影响排队和运行中的任务。

3. 回收站生命周期。
   - 恢复时使用统一冲突策略。
   - 恢复时已按当前挂载快照重新解析原虚拟路径，并要求目标父目录仍处于可写挂载中。
   - 恢复冲突策略已补充 API 冒烟覆盖：默认自动重命名、显式 `reject` 保留回收站记录、显式 `overwrite` 替换目标文件。
   - 增加按天数和总大小清理。
   - 已支持显式请求清理和列表请求的间隔触发清理；启动时只加载索引，不默认扫描回收站内容。
   - 普通文件删除时已记录 `sizeBytes`，容量清理优先使用记录大小；文件夹不为了统计而预扫，缺少大小时才按需计算。
   - 暂不增加后台定时器，避免为轻量项目引入常驻扫描复杂度。

4. 轻量搜索。
   - 默认启动时不扫描所有挂载目录。
   - 保留手动重建索引。
   - 手动重建已支持取消，扫描会在目录和条目边界检查取消状态。
   - 对本应用产生的变更做增量更新。
   - 增量更新已覆盖创建、保存、上传、移动、删除、恢复和后台任务输出的根路径；文件夹复制和解压不会为了索引递归扫描全部子项。
   - 挂载创建、更新和删除会清理相关挂载的内存索引，不自动扫描新挂载目录。
   - 搜索查询已改为单次遍历并只克隆当前页结果，不再先收集全部匹配项再分页。
   - 第一阶段只搜索文件名、扩展名和路径。

5. 审计和指标。
   - 默认审计登录、退出、上传、保存、重命名或移动、删除、恢复和重要任务事件。
   - 默认不审计目录列表、预览和下载。
   - 审计日志使用 JSONL，并增加轮转或保留策略。
   - 已增加 `WEB_FILE_BROWSER_AUDIT_MAX_BYTES`，默认 10 MiB，达到上限后在下一次审计写入前轮转。
   - 已增加 `WEB_FILE_BROWSER_AUDIT_RETENTION_FILES`，默认保留 8 个旧轮转文件，只在新轮转后触发清理。
   - 已增加 `POST /api/audit/cleanup`，可主动清理旧轮转审计文件；启动时仍不扫描审计目录。
   - 已补充轻量指标快照：活动传输、活动目录扫描、单 IP 并发占用、任务状态汇总、回收站条目数量和索引状态。
   - 暂不递归估算回收站总大小，避免指标接口触发额外磁盘遍历。
   - 已拆分存活检查和就绪检查：`/api/health` 只看进程响应，`/api/ready` 检查认证初始化和必要运行目录。
   - 下载接口已确认不写审计日志，避免频繁下载带来额外 JSONL 写入。

6. 错误响应和运维文档。
   - API 错误响应统一为 `{ "code": "...", "message": "..." }`。
   - 稳定错误码、典型场景和前端处理建议已写入 `docs/API_ERRORS.md`。
   - `src/error.rs` 已补充错误 code 与 HTTP 状态的单元测试。
   - 新增或调整错误 code 时，需要同步更新文档和测试。

## 保持简单或不做

- 不做多用户账号、角色、用户组或分享链接。
- 不做公网产品能力。
- 不做云同步。
- 不做 WebDAV、SFTP 或 SMB 协议服务器。
- 不做全文搜索、OCR 或媒体元数据索引。
- 不把缩略图生成作为基础功能。
- 不做视频转码、Office 转换或 PDF 转换。
- 不做实时协同编辑。
- 不做插件体系。
- 不做分布式任务，也不做进程重启后的任务恢复。

## 最近推进

- 内容接口已支持 ETag 和 `If-Match` 保存保护，缺少版本标记返回 428，版本过期返回 412。
- 搜索索引已禁止默认启动全量扫描，只有显式开启启动重建开关才会扫描挂载目录。
- `/api/file/*` 元数据响应已复用同一次 JSON 序列化结果生成 ETag 和响应体，`Last-Modified` 直接从元数据结构计算，避免大目录响应双重序列化和 JSON 反解析。
- `/api/file/*` 元数据条件请求已支持 `If-None-Match` 和 `If-Modified-Since`，缓存命中时返回 `304`；默认目录响应会用目录自身 mtime 生成 `Last-Modified`，且 `If-Modified-Since` 命中时可跳过 JSON 序列化。
- `detail=basic` 元数据请求已增加轻量条件请求预检查：没有 `If-None-Match` 时，先用目标自身 mtime 判断 `If-Modified-Since`，命中可在占用目录扫描并发许可前返回 `304`。
- 目录按名称排序的轻量分页读取已改为只保留当前页排序窗口，避免大目录请求把页外条目全部留在内存中。
- 目录总数按需返回已补充 API 冒烟测试：默认分页响应不包含 `folderTotal/fileTotal`，只有 `includeTotal=true` 时才返回总数，避免路由层回归到默认统计大目录。
- 目录分页上限已补充 API 冒烟测试：请求 `limit` 会被 `WEB_FILE_BROWSER_MAX_DIR_PAGE_SIZE` 截断，`limit=0` 返回稳定 JSON 错误，避免脚本或前端触发过大的单次目录响应。
- 目录昂贵排序已补充 API 冒烟测试：`detail=basic` 禁止按大小或修改时间排序，必须显式使用 `detail=full`，避免轻量请求因排序参数隐式对全部条目执行 `stat`。
- 现有写入口已接入冲突策略：新建、上传、移动、复制、回收站恢复默认 `autoRename`，并支持请求或配置显式选择 `reject`、`overwrite`；同步和异步文件覆盖替换已改为先保护旧目标，避免替换失败时先删除旧文件。
- 在线编辑安全限制已完成基础实现：编辑器读取使用 `mode=edit`，保存和读取都会受编辑大小上限保护，二进制文件默认拒绝在线编辑。
- 在线编辑白名单已完成基础实现：可通过 `WEB_FILE_BROWSER_EDITABLE_EXTENSIONS` 和 `WEB_FILE_BROWSER_EDITABLE_MIME_TYPES` 收紧允许编辑的文件类型，未配置时保持现有文本判断。
- 普通 `HEAD /api/content/*` 和 `HEAD /api/download/*` 已优化为只读取文件元数据，不打开文件句柄；编辑模式 HEAD 仍保留文本安全检查。
- 内容和下载 HEAD、Range HEAD 以及非法 Range 已补充 API 冒烟测试：HEAD 只返回头信息和空正文，非法 Range 返回稳定 `416 RANGE_NOT_SATISFIABLE`。
- 单管理员密码修改已完成基础实现：修改成功后旧会话失效，当前请求获得新会话；服务端会话已与 Cookie 对齐为 7 天有效，并会懒清理过期项。
- 默认 CORS 已收紧：默认同源，跨域必须显式配置可信来源，且不支持 `*`。
- Docker 部署材料已新增：Dockerfile、`.dockerignore`、Compose 示例、`env.example` 和 Linux Docker 部署文档。
- Docker 功能冒烟已在真实 Linux Docker 环境通过：`scripts/docker-smoke.sh` 使用 `.smoke/docker` 临时目录，覆盖前端静态托管、登录、挂载、编辑保存、下载、上传、zip/tar.gz 压缩和解压、删除/恢复和指标接口。
- Docker 性能冒烟脚本已新增：`scripts/docker-perf-smoke.sh` 默认验证 1 万项目录分页、按需总数、64 MiB 上传下载、Range、大文件编辑保护、16 MiB tar.gz 压缩解压、回收站恢复和指标接口。
- Linux Docker 冒烟踩坑已固化到部署文档：Yarn 4 构建需提前复制 `.yarnrc.yml`，运行镜像不再安装 `curl`，UID/GID 直接用数字用户，跨挂载回收站恢复使用复制后清理兜底。
- 压缩/解压后台任务已完成基础实现：`tar.gz`、`tgz`、`zip` 按块读写，输出先写临时路径，完成后移动到最终路径，解压会拒绝越界路径和不支持条目。
- 压缩/解压大文件 API 冒烟已补充：zip 和 tar.gz 都使用超过单块缓冲区的文件验证任务字节进度和解压后内容一致。
- 主界面已补充压缩/解压最小入口：工具栏和右键菜单可创建后台任务，任务完成后刷新目录查看结果。
- 主界面已补充后台任务面板：可手动刷新、查看任务状态、进度、速度、错误数量，并可取消排队或运行中的任务。
- 后台任务历史已增加轻量内存清理：默认只保留最近 200 条已结束任务，避免长期运行后任务列表和指标无限增长；`POST /api/tasks/cleanup` 可主动清理已结束任务。
- 后台任务创建接口已补充空路径和输出名拦截：复制、移动、删除、压缩和解压在创建任务前拒绝空字符串路径，压缩 `outputName` 和解压 `folderName` 也会提前校验，避免明显无效请求进入任务列表或后台调度。
- 审计 JSONL 已增加按大小轮转和数量保留：默认单文件 10 MiB，设置 `WEB_FILE_BROWSER_AUDIT_MAX_BYTES=0` 可关闭轮转；默认保留 8 个旧轮转文件，清理只在新轮转或显式调用 `POST /api/audit/cleanup` 后发生；轮转判断已改为启动读取一次已有大小，后续使用进程内累计大小。
- 回收站保留策略已支持请求触发：启动时只加载索引，不默认扫描回收站内容；`POST /api/trash/cleanup` 会按天数和容量清理，`GET /api/trash` 会按间隔触发一次保留检查。
- 回收站文件记录已增加低成本大小字段：普通文件删除时记录 `sizeBytes`，容量清理优先使用索引大小，减少清理时对文件记录的磁盘遍历。
- 回收站恢复已重新接入当前挂载边界：恢复前会按原虚拟路径解析当前父目录，并检查挂载仍可写，避免旧真实路径绕过当前配置。
- 回收站恢复冲突策略已补充 API 冒烟测试：默认自动重命名、显式 `reject` 保留记录和目标文件、显式 `overwrite` 替换目标文件。
- 后台任务取消响应已增强：压缩、解压和复制在更多文件系统边界检查取消状态，限速等待也会分片检查取消；移动任务在真实移动前检查取消；复制取消会清理临时输出并保留旧目标文件；删除任务已移除目录大小预扫。
- 后台复制任务已在用户选择的源路径和递归目录条目两层使用 `symlink_metadata` 分类，拒绝符号链接和不支持的特殊文件，避免跟随链接造成越界或循环。
- 移动和删除已拒绝直接操作符号链接，避免最终路径解析跟随链接后误操作真实目标。
- 复制、压缩和解压已统一使用批量进度上报：小块读写不会每次跨线程更新任务锁，按字节阈值或时间间隔合并上报，成功收尾前刷新精确字节数，同时保持低延迟取消检查。
- 已结束后台任务不再接受取消请求：`POST /api/tasks/{id}/cancel` 对 completed/failed/cancelled 任务返回 `409 CONFLICT`，避免出现状态已完成但 `cancelled=true` 的混合状态。
- `/api/metrics` 已补充轻量内存指标快照：并发占用、任务状态分布、回收站条目数和索引状态。
- `/api/metrics` 已补充 API 冒烟测试：验证映射、会话、回收站、限流、任务和索引状态来自轻量运行时快照，不依赖递归磁盘扫描。
- 请求并发限制已补充服务层保护测试：目录扫描、文件传输和单 IP 并发达到上限时会立即返回 `429 TOO_MANY_REQUESTS`，不在服务端排队等待。

- 流式读取和下载的传输并发许可已绑定到响应体生命周期：大文件响应未消费或未丢弃前会持续占用 `activeTransfers`，后续传输请求会按上限返回 `429 TOO_MANY_REQUESTS`。

- `/api/ready` 文件系统探测已移到阻塞线程池，避免 Docker 健康检查在异步运行时里执行同步 I/O；成功和管理员密码未初始化场景已补充 API 冒烟测试。
- API 错误码文档已新增：`docs/API_ERRORS.md` 记录稳定 code、典型场景和前端处理建议；`src/error.rs` 已补充错误响应 code/status 单元测试。
- 后台任务状态已补充当前处理条目：任务面板可显示 `currentPath`，任务结束后自动清空，避免展示过期状态。

- 挂载配置变化后会按挂载点清理内存索引，避免旧映射结果残留，同时不触发自动重建扫描。
- 搜索查询分页已减少临时分配：`/api/search` 统计总数时只克隆当前页结果，`/api/recent` 不再克隆整个索引后排序。
- 搜索索引条目已缓存小写虚拟路径：文件名和扩展名天然包含在虚拟路径中，查询不再为每个条目重复生成名称、路径和扩展名的小写副本，移动或重命名时同步刷新缓存。
- 最近文件查询已改为固定窗口选择：只保留 `limit` 个候选项，并使用索引条目缓存的数值修改时间排序，避免大索引下为了少量最近结果全量排序索引引用或按字符串误排时间戳。
- 搜索和最近文件接口已补充分页上限保护：请求 `limit` 会被 `WEB_FILE_BROWSER_MAX_DIR_PAGE_SIZE` 截断，`limit=0` 返回稳定 JSON 错误，避免错误参数触发过大的响应或被静默修正。
