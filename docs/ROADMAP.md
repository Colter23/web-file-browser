# Web File Browser 路线图

这份路线图是项目的长期任务列表，必须保持在 [PROJECT_BOUNDARY.md](PROJECT_BOUNDARY.md) 描述的边界内。

项目应逐步推进。每个阶段都要保持后端轻量、局域网取向、单管理员、适合 Linux Docker 部署，并且性能优先。

## 指导决策

- 目标部署环境是 Linux Docker。
- 目标目录规模约为单目录 1 万个条目。
- 更大目录尽量可用，但不为了极端规模引入复杂架构。
- 只保留单管理员。
- 项目注释、日志、错误提示、前端提示文本和管理文案默认使用中文。
- 后续改动触及旧英文注释或提示文案时，应顺手改为中文。
- 开发期不要求兼容旧接口；模型或接口不合理时优先破坏式重构。
- 破坏式重构要同步收敛前端调用、测试和文档，不为了旧接口形状长期保留适配层。
- 后端由本线程推进，前端由另一个 AI 推进；后端阶段默认交付稳定 API 契约、错误码和对接说明，不主动扩张前端 UI 工作。
- 文件内容和大操作必须保持流式。
- 基础配置继续使用 JSON。
- 搜索索引可以先保留在内存中，只有确有需要时再引入 SQLite。
- 默认启动时不全量扫描挂载目录。
- 文件操作冲突必须可配置，默认自动重命名。
- 默认不审计下载、目录列表和预览/读取。
- 暂不做文件夹实时打包下载；压缩/解压走后台任务。
- 默认不信任代理来源头；只有服务位于可信反向代理之后时才显式启用 `X-Forwarded-For`。

## 第一阶段：稳定核心文件管理

目标：让当前常用流程更安全、更省资源，不扩张产品面。

任务：

- 目录总数统计可选。
  - 默认目录响应不做昂贵的总数统计。
  - 增加 `includeTotal=true` 给确实需要总数的客户端。
  - `hasMore` 保持按分页结果低成本计算。
- 可配置冲突策略。
  - 全局默认 `autoRename`。
  - 请求级支持 `autoRename`、`reject` 和显式 `overwrite`。
  - `overwrite` 不能成为意外默认值。
  - 统一应用到上传、新建、复制、移动、压缩输出和解压输出。
  - 文件覆盖替换尽量先保护旧目标，替换失败时不先删除旧文件。
- 文件编辑冲突保护。
  - `GET` 和 `HEAD /api/content/{path...}` 返回稳定 ETag 或版本标记。
  - `PUT /api/content/{path...}` 支持 `If-Match`。
  - 文件被外部修改时保存失败，并返回清晰错误。
  - 已完成基础实现。
- 在线编辑安全限制。
  - 可配置最大可编辑文件大小。
  - 默认不把二进制文件当作文本处理。
  - 已完成基础实现。
  - 可编辑扩展名或 MIME 分组配置已完成基础实现。
- 停止启动搜索扫描。
  - 即使启用搜索，也不应默认启动就遍历所有挂载目录。
  - 保留手动重建和后续增量更新。
- 收紧默认 CORS。
  - 默认同源。
  - 开发或可信局域网跨域通过配置显式开启。
  - 已完成基础实现。
- 管理员密码修改 API。
  - 继续保持单管理员。
  - 不引入多用户或角色模型。
  - 已完成基础实现。

- `yarn.cmd build`
- API 冒烟覆盖目录分页、冲突策略、保存 ETag 冲突、上传和新建。

## 第三阶段：压缩和解压

目标：支持常用压缩流程，同时保持轻量和安全。

任务：

- 增加 `POST /api/tasks/archive`。
  - 输入：源路径、目标路径、格式、可选输出名称、冲突策略。
  - 支持格式：`tar.gz`、`zip`。
  - 默认压缩等级偏向速度。
  - 读写保持流式。
  - 遵守只读挂载检查。
- 增加 `POST /api/tasks/extract`。
  - 输入：压缩包路径、目标路径、冲突策略。
  - 支持格式：`tar.gz`、`zip`。
  - 解压保持流式。
  - 遵守只读挂载检查。
- 增加压缩/解压安全限制。
  - 最大解压字节数。
  - 最大解压文件数。
  - 限制最大路径深度，默认 64 层，可通过环境变量调整。
  - 拒绝路径穿越和 Zip Slip。
  - 拒绝不支持或可疑条目。
- 扩展任务状态。
  - 当前条目。
  - 已处理字节。
  - 错误。
  - 取消状态。

当前状态：

- 后端基础实现已完成。
- 压缩支持 `tarGz` 和 `zip`，写入同目录临时文件后再移动或替换。
- 压缩源路径和递归条目都会拒绝符号链接和不支持的特殊文件。
- 解压支持 `zip`、`tar.gz` 和 `tgz`，默认在目标目录下创建压缩包同名文件夹。
- 解压已接入路径穿越、Zip Slip、符号链接、不支持条目类型、最大解压字节数、最大解压条目数和最大路径深度保护。
- 已补充最小前端入口和任务状态面板；后续仍需真实 Linux Docker 冒烟。

暂不做：

- 实时“下载文件夹为 zip”。
- `rar`、`7z`、加密压缩包、分卷压缩包。
- 压缩包预览 UI。

验证：

- 单元测试覆盖冲突策略。
- API 冒烟已覆盖 zip 压缩/解压和 tar.gz 压缩/解压。

最近完成：

- 内容接口 ETag 和 `If-Match` 保存保护：读取/HEAD 返回 ETag，保存缺失版本返回 428，版本过期返回 412。
- 搜索索引启动策略：启用搜索不再自动全量扫描，启动重建需要显式配置。
- 现有写入口的冲突策略：新建、上传、移动、复制、回收站恢复默认自动重命名，并支持显式拒绝或覆盖；回收站恢复已补充 `reject` 和 `overwrite` 的 API 冒烟覆盖；同步和异步覆盖替换已改为先保护旧目标。
- 目录 `includeTotal=true` 和更低成本的默认分页。
- 目录/文件元数据响应复用同一次 JSON 序列化结果生成 ETag 和响应体，`Last-Modified` 直接从元数据结构计算，避免大目录响应双重序列化和 JSON 反解析。
- 目录/文件元数据条件请求：`If-None-Match` 和 `If-Modified-Since` 命中时返回 `304`；默认目录响应复用目录自身 mtime 生成 `Last-Modified`，没有 `If-None-Match` 且 `If-Modified-Since` 命中时跳过 JSON 序列化，减少重复传输且不额外 `stat` 子项。
- basic 条件请求预检查：`detail=basic` 且没有 `If-None-Match` 时，会先用目标自身 mtime 判断 `If-Modified-Since`，命中可避免占用目录扫描并发许可和扫描大目录。
- 目录名称排序分页优化：轻量读取只保留当前页所需排序窗口，减少大目录页外条目的内存占用。
- 目录总数按需返回：API 冒烟已覆盖默认分页响应不包含 `folderTotal/fileTotal`，只有 `includeTotal=true` 时才返回总数，防止默认请求触发大目录总数统计。
- 目录分页上限：API 冒烟已覆盖请求 `limit` 被 `WEB_FILE_BROWSER_MAX_DIR_PAGE_SIZE` 截断，`limit=0` 返回稳定 JSON 错误，防止单次目录响应过大。
- 目录昂贵排序：API 冒烟已覆盖 `detail=basic` 禁止按大小或修改时间排序，显式 `detail=full` 后才允许执行需要条目元数据的排序。
- 在线编辑安全限制：编辑器读取使用 `mode=edit`，保存和读取都会受编辑大小上限保护，二进制文件默认拒绝在线编辑。
- 在线编辑白名单：可通过环境变量配置允许编辑的扩展名或 MIME 分组，默认不配置时保持现有自动文本判断。
- 传输 HEAD 探测：普通 `HEAD /api/content/*` 和 `HEAD /api/download/*` 已改为只读取元数据，不打开文件句柄；API 冒烟已覆盖 HEAD 空正文、Range HEAD 和非法 Range 的 `416 RANGE_NOT_SATISFIABLE`。
- 单管理员密码修改：登录后可修改密码，修改成功后旧会话失效，当前请求获得新会话；服务端会话与 Cookie 对齐为 7 天有效，并在登录、鉴权和指标统计时懒清理。
- 默认 CORS：默认同源，跨域必须显式配置可信来源，且不支持 `*`。
- Docker 部署材料：已新增 Dockerfile、`.dockerignore`、Compose 示例、`env.example` 和 Linux Docker 部署文档。
- Docker 冒烟脚本：已新增 `scripts/docker-smoke.sh`，覆盖前端静态托管、登录、挂载、编辑保存、下载、上传、zip/tar.gz 压缩和解压、删除/恢复和指标接口；本地 Windows 环境没有可用 Docker，仍需在真实 Linux Docker 环境执行。
- 压缩/解压后台任务：已新增 `POST /api/tasks/archive` 和 `POST /api/tasks/extract`，支持 `tar.gz`、`tgz`、`zip`，并接入路径安全和可选解压上限。
- 压缩/解压大文件验证：zip 和 tar.gz API 冒烟已使用超过单块缓冲区的文件验证任务字节进度和解压内容一致。
- 压缩/解压前端入口：主界面工具栏和右键菜单已能创建后台压缩/解压任务。
- 后台任务面板：主界面已能手动刷新任务、查看状态/进度/速度/错误数量，并取消排队或运行中的任务。
- 后台任务历史清理：内存中默认只保留最近 200 条已结束任务，设置页会展示当前保留上限；`POST /api/tasks/cleanup` 可主动清理已结束任务，不影响排队或运行中的任务。
- 后台任务创建参数：复制、移动、删除、压缩和解压会在创建任务前拒绝空字符串路径，压缩 `outputName` 和解压 `folderName` 也会提前校验，避免明显无效的请求进入后台队列。
- 审计 JSONL 轮转和保留：默认单文件 10 MiB、保留 8 个旧轮转文件；启动时读取一次当前文件大小，后续通过进程内累计大小判断是否轮转，旧轮转清理只在发生新轮转或显式调用 `POST /api/audit/cleanup` 后执行。
- 回收站保留清理：启动时只加载回收站索引，不默认扫描回收站内容；`POST /api/trash/cleanup` 显式清理，`GET /api/trash` 按间隔触发一次保留检查。
- 回收站容量清理优化：普通文件删除时记录 `sizeBytes`，容量策略优先使用索引大小，减少清理时对文件记录的磁盘访问。
- 回收站恢复边界：恢复前按当前挂载快照重新解析原虚拟路径，并要求目标父目录仍在可写挂载中。
- 后台任务取消响应：压缩、解压和复制增加更多取消检查点，限速等待也会分片检查取消；移动任务在真实移动前检查取消；复制取消会清理临时输出并保留旧目标文件，删除任务不再为了进度字节预扫目录。
- 后台复制安全边界：用户选择的源路径和递归目录条目都会使用 `symlink_metadata` 分类，拒绝符号链接和不支持的特殊文件，避免跟随链接越界或陷入循环。
- 移动/删除安全边界：直接选择符号链接作为源路径或删除目标会失败，不会跟随链接去移动或删除真实目标。
- 后台任务进度上报：复制、压缩和解压已统一合并小块进度更新，按字节阈值或时间间隔写入内存任务状态，减少大文件操作中的跨线程锁竞争，并在成功收尾前刷新精确字节数。
- 指标快照：`GET /api/metrics` 已返回并发占用、任务状态分布、回收站条目数和索引状态，且不触发磁盘目录遍历。
- 指标 API 冒烟：已覆盖映射、会话、回收站、限流、任务和索引状态字段，保护 `/api/metrics` 继续作为轻量运行时快照。
- 请求并发限制：目录扫描、文件传输和单 IP 并发达到上限时直接返回 `429 TOO_MANY_REQUESTS`，服务端不为了过量请求排队等待。
- 单 IP 并发保护 API 冒烟：已用挂起上传请求验证受保护路由会按来源 IP 拒绝后续并发请求；默认不信任代理头时，伪造不同 `X-Forwarded-For` 不能绕过连接 IP 限流。
- 流式传输限流：读取和下载响应体会持有传输并发许可，直到响应体消费结束或被丢弃，避免大文件下载期间限流提前释放。
- 单 IP 并发保护：空闲 IP 记录会在请求和指标路径被轻量清理，`trackedIps` 只反映仍有活跃请求的限流记录。
- 就绪检查：`GET /api/ready` 已拆分自 `/api/health`，用于 Docker 健康检查，不扫描挂载目录。
- 就绪检查 I/O 隔离：`/api/ready` 的文件系统探测已移到阻塞线程池，成功和管理员密码未初始化场景已有 API 冒烟保护。
- 错误码文档：`docs/API_ERRORS.md` 已记录统一错误响应、稳定 code、典型场景和前端处理建议，并用单元测试保护 `AppError` 的 code/status 映射。
- 后台任务当前条目：`TaskStatus.currentPath` 已用于展示复制、移动、删除、压缩和解压的当前处理目标，任务结束后清空。
- 搜索增量更新：创建、保存、上传、移动、删除、恢复和后台任务输出会更新内存索引；索引关闭时不做额外工作。
- 挂载索引清理：挂载创建、更新和删除会移除相关挂载的内存索引，不触发自动重建。
- 搜索查询分页优化：搜索结果不再先全量克隆匹配项，最近文件也不再先克隆整个索引。
- 搜索大小写匹配优化：索引写入时缓存小写虚拟路径，查询直接复用缓存，避免每次搜索为所有候选条目重复分配名称、路径和扩展名的小写字符串。
- 最近文件查询优化：只维护 `limit` 大小的候选窗口，并使用缓存的数值修改时间排序，避免大索引下为了少量最近结果全量排序索引引用或按字符串误排时间戳。
- 搜索分页上限：`/api/search` 和 `/api/recent` 会截断过大的 `limit`，并对 `limit=0` 返回稳定 JSON 错误，防止脚本或前端错误参数制造异常响应。
- 搜索重建取消：`POST /api/index/cancel` 已可取消正在进行的手动或启动重建。
- 搜索索引错误状态：索引未启用时重建和取消会返回稳定 `400 BAD_REQUEST`，索引空闲时取消会返回稳定 `409 CONFLICT`，避免前端把无效操作误判为后台任务。
- 搜索重建重复请求：`POST /api/index/rebuild` 在当前状态为 `scanning` 时直接返回 `409 CONFLICT`，并由 API 冒烟测试保护，避免重复重建请求被误报为已接受。
- API 冒烟测试：已覆盖公开健康检查、受保护接口未登录拦截、登录后挂载创建、文件元数据读取、Range 内容读取、只读挂载写入失败、路径越界拦截、跨挂载移动拒绝、后台移动任务取消、后台复制到只读挂载失败、后台复制任务取消、后台复制任务的部分成功和错误列表、任务显式清理、zip/tar.gz 压缩解压任务链路、zip/tar.gz 大文件压缩解压字节进度、搜索重建/取消/最近文件/增量更新链路、审计轮转清理，以及回收站删除/列表/恢复/永久删除/清空/显式清理链路。
- 上传和保存大小上限：API 冒烟已覆盖超过 `WEB_FILE_BROWSER_MAX_UPLOAD_BYTES` 时返回 `413 PAYLOAD_TOO_LARGE`，并确认失败不会改写原文件或留下上传目标。
- 下载审计边界：`/api/download/*` 默认不再写入审计日志，避免频繁下载制造额外磁盘写入。
- 审计 API 边界：已覆盖登录、保存、上传、删除、回收站恢复会写入审计，目录列表、内容读取和下载不会写入审计。
