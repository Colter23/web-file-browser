# Web File Browser 项目边界

## 产品定位

Web File Browser 是一个面向局域网的轻量单管理员文件管理器。

后端优先级依次是性能、低资源占用、部署简单、文件系统行为可预测。除非明确改变方向，后续开发都应保持在这个边界内。

## 语言和重构约定

- 项目文档、代码注释、后端日志、错误提示、前端提示性文本和管理界面文案默认使用中文。
- 新增或修改代码时，如果触及旧的英文注释、提示文案、日志或错误信息，应一并改为中文。
- API 路径、环境变量、协议字段、Rust/TypeScript 标识符可以继续使用英文，以保持工程表达清晰。
- 当前仍处于开发期，不为历史接口兼容性牺牲模型和接口设计。
- 如果现有模型、路由或数据结构不合理，优先做破坏式重构，直接收敛到更清晰、更高性能的形态。
- 破坏式重构应同步调整前端调用、测试和文档，不保留只为兼容旧接口存在的转换层。
- 后续开发评估方案时，兼容旧行为的成本只有在确实影响当前常用流程时才纳入考虑。

## 协作分工

- 后端开发由本线程负责，默认工作范围包括 Rust 后端、API 模型、接口文档、部署脚本、后端测试和性能验证。
- 前端开发由另一个 AI 负责，本线程默认不主动修改 `ui/` 下的页面、组件和交互实现。
- 后端如果进行破坏式接口调整，必须同步更新 README、边界文档或 API 文档，清楚说明请求字段、响应字段、错误码和前端需要调整的调用点。
- 只有在用户明确要求、后端接口变更必须立刻修通现有构建，或需要提供最小对接示例时，本线程才修改前端代码。
- 后端输出给前端协作者的信息应以稳定 API 契约为主，避免把内部实现细节变成前端依赖。

## 目标环境

- 主要部署环境是 Linux Docker。
- 前端由 Rust 后端作为静态文件托管。
- 运行时数据、配置、回收站、审计日志和挂载目录都应通过卷挂载管理。
- Docker 文档需要说明 UID/GID、文件权限、只读挂载、端口和环境变量。
- Docker 功能冒烟和性能冒烟需要保持可执行；性能冒烟默认基线是 1 万项目录和 64 MiB 大文件流式传输，已在真实 Linux Docker 环境通过。
- 应用不能依赖桌面环境或系统回收站。

## 核心范围

项目需要覆盖这些常用局域网文件管理流程：

- 单管理员登录。
- 将本地或服务器目录挂载到虚拟路径。
- 分页浏览目录。
- 上传和下载文件。
- 预览或读取文件内容。
- 编辑并保存文本文件。
- 新建文件和文件夹。
- 重命名和移动条目。
- 删除条目到应用自管回收站。
- 恢复或永久删除回收站条目。
- 将大操作放入后台任务。
- 后台复制、移动和删除。
- 后台压缩和解压文件或文件夹。
- 提供轻量文件名和路径搜索。
- 提供健康检查和指标接口。

## 性能规则

- 文件内容传输必须全程流式。
- 上传、下载、预览、保存、复制、压缩和解压都不能把大文件整体读入内存。
- 目录接口默认分页并返回轻量元数据。
- 目录浏览主要按 1 万个条目设计。
- 更大的目录尽量保持可用，但不为了极端目录强行引入复杂架构。
- 默认目录列表避免不必要的 `stat` 调用。
- 目录总数只有在请求方需要时才计算。
- 默认不对文件传输响应做 gzip 或 brotli 压缩。
- Range 请求只支持单段，除非以后有明确需求。
- 后台任务必须遵守并发限制和可选限速。
- 避免启动时无意义扫描挂载磁盘。

## 目录元数据边界

- 默认目录详情为 `basic`。
- `basic` 尽量只返回名称、类型和路径。
- size、modified 等完整元数据只在 `detail=full` 时返回。
- 按大小或修改时间排序可以要求 `detail=full`。
- 隐藏文件可见性应支持配置或请求控制。
- 元数据响应应支持 `ETag`、`Last-Modified`、`If-None-Match` 和 `If-Modified-Since`，缓存命中时返回 `304`。
- 默认 `detail=basic` 的真实目录 `Last-Modified` 使用目录自身修改时间，不为了响应头额外 `stat` 子项。
- 没有 `If-None-Match` 且 `If-Modified-Since` 已命中时，应尽量在序列化目录 JSON 之前返回 `304`。
- 主要目标是让 1 万项目录稳定可用。

## 冲突策略

文件操作冲突必须可配置。

默认行为：

- 同名时自动重命名。
- 示例：`a.txt` -> `a (1).txt`。

允许的显式模式：

- 目标存在时拒绝。
- 仅在明确请求或配置时覆盖。

覆盖不能成为意外默认值。
文件覆盖替换应尽量先保留旧目标，只有新文件成功落到目标路径后才清理旧目标；替换失败时应保留旧目标和新源文件。

## 编辑边界

后端已经通过内容接口支持文件编辑：

- `GET /api/content/{path...}` 读取文件内容。
- `GET /api/content/{path...}?mode=edit` 作为编辑器读取入口，会拒绝超出编辑上限或看起来不是文本的文件。
- `PUT /api/content/{path...}` 通过临时文件写入和原子替换保存文件内容。
- 内容接口返回 ETag 或等价版本标记。
- 保存接口支持 `If-Match`，避免浏览器旧内容静默覆盖外部修改。
- 在线编辑已有可配置最大文件大小限制。
- 在线编辑已有文本/二进制安全判断。

仍在范围内但不是当前必需：

- 已支持可编辑扩展名或 MIME 分组配置；默认不配置时仍按文本采样判断。
- 后续可增加更细的字符编码提示。

不做：

- 实时协同编辑。
- 富文本或复杂文档编辑。
- Office 或 PDF 转换。

## 压缩和解压边界

压缩和解压属于范围内功能，但只作为轻量后台任务实现。

范围内：

- `POST /api/tasks/archive`。
- `POST /api/tasks/extract`。
- 压缩和解压任务可取消。
- 优先支持 Linux/Docker 友好的 `tar.gz`。
- 支持常用 `zip`。
- 文件读写保持流式。
- 使用偏向速度的默认压缩等级，不追求最高压缩率。
- 遵守可写挂载检查和冲突策略。

必要安全规则：

- 防止路径穿越和 Zip Slip。
- 不允许解压到目标可写挂载点之外。
- 压缩源路径和递归条目都拒绝符号链接和不支持的特殊文件。
- 支持可选最大解压字节数。
- 支持可选最大解压文件数。
- 支持可配置路径深度限制，默认最多 64 层。
- 拒绝不支持或可疑的压缩条目。

暂不做：

- 实时“下载文件夹为 zip”。
- `rar`、`7z`、加密压缩包、分卷压缩包。
- 压缩包预览 UI。

## 搜索边界

搜索保持轻量。

范围内：

- 搜索文件名、扩展名和路径。
- 手动重建索引。
- 显式启用时低速后台扫描。
- 对本应用产生的文件变更做增量更新。

默认行为：

- 不在每次启动时全量扫描挂载目录。
- 避免无意义磁盘遍历。

存储：

- JSON 继续作为基础配置存储。
- 轻量版本可以使用内存搜索索引。
- SQLite 以后只在搜索索引、审计历史、任务历史这类增长型数据确有需要时引入。

不做：

- 全文搜索。
- OCR。
- 媒体元数据索引。
- 将复杂文件系统监听作为基础要求。

## 审计边界

默认审计保持轻量。

默认审计：

- 登录。
- 退出。
- 上传。
- 保存。
- 重命名或移动。
- 删除。
- 恢复。
- 后台任务创建和重要结果。

默认不审计：

- 每次下载。
- 每次目录列表。
- 每次预览或读取。

原因是这些操作在局域网使用中可能制造大量日志 I/O，但价值有限。

审计存储：

- 默认使用 append-only JSONL。
- 在日志过大前增加大小轮转或保留清理。
- 默认按 `WEB_FILE_BROWSER_AUDIT_MAX_BYTES` 做单文件大小轮转，默认 10 MiB。
- 设置 `WEB_FILE_BROWSER_AUDIT_MAX_BYTES=0` 可以关闭轮转。
- 默认最多保留 8 个审计轮转文件，可通过 `WEB_FILE_BROWSER_AUDIT_RETENTION_FILES` 调整。
- 审计保留清理只在发生新轮转后执行，不在启动时扫描审计目录。
- `POST /api/audit/cleanup` 可主动清理旧轮转审计文件，并记录一条轻量审计事件。
- SQLite 以后可选，不是当前必需。

## 认证和访问边界

- 保持单管理员。
- 不增加用户、角色、用户组、分享链接或公开链接。
- 管理员登录后可以修改密码。
- 服务端会话默认 7 天有效，与 Cookie 生命周期一致；登录、鉴权和指标统计时懒清理过期会话，不增加后台定时器。
- 忘记密码恢复可以继续依赖手动或环境变量方式。
- 默认 CORS 使用同源策略。
- 跨域访问必须通过 `WEB_FILE_BROWSER_CORS_ORIGINS` 为开发环境或可信局域网显式配置。
- CORS 不支持 `*`，必须列出具体可信来源。
- 默认不信任客户端传入的 `X-Forwarded-For`；只有服务位于可信反向代理之后时，才通过 `WEB_FILE_BROWSER_TRUST_PROXY_HEADERS` 显式启用代理来源头。

## 回收站边界

应用使用自管回收站。

范围内：

- 删除时移动到回收站。
- 删除到回收站时，优先移动到挂载根目录下的 `.web-file-browser-trash/`，减少 Linux Docker 跨挂载复制；挂载内回收站不可用时再回退到中央 `data/trash/`。
- 支持管理员显式永久删除原文件，默认删除仍进入回收站。
- 记录原路径、删除时间、操作者和条目类型。
- 恢复回收站条目。
- 批量恢复回收站条目。
- 恢复必须按当前挂载配置重新解析原虚拟路径，并遵守可写挂载检查。
- 永久清理条目。
- 批量永久清理回收站条目。
- 清空回收站。
- 按天数和总大小保留。
- 普通文件删除时记录 `sizeBytes`，容量清理优先使用记录大小；文件夹不为了容量统计提前递归扫描。
- 周期性或请求触发清理，不在启动时默认扫描回收站内容。

不做：

- 系统回收站集成。
- 多用户回收站归属。

## 后台任务边界

范围内：

- 复制。
- 移动。
- 删除。
- 压缩。
- 解压。
- 取消。
- 进度。
- 已处理字节。
- 速度。
- 错误列表。
- 部分成功报告。
- 并发和限速。

默认存储：

- 内存任务状态足够。
- 只保留最近任务历史。
- 已结束任务默认只保留最近 200 条，可通过 `WEB_FILE_BROWSER_TASK_HISTORY_LIMIT` 调整。
- 排队和运行中的任务不因历史清理被移除。
- `POST /api/tasks/cleanup` 可主动清理已结束任务历史，不影响排队和运行中的任务。

除非明确需要，否则不做：

- 进程重启后的任务恢复。
- 分布式 worker。
- 复杂调度。

## 明确不做

- 公网部署作为产品目标。
- 多用户权限。
- 基于角色的访问控制。
- 分享链接。
- 云同步。
- WebDAV/SFTP/SMB 协议服务器。
- 全文索引。
- 缩略图生成作为基础功能。
- 视频转码。
- Office/PDF 转换。
- 实时协作。
- 大型企业审计系统。
- 复杂插件体系。

## 近期后端优先级

1. 根据 `docs/API_CONTRACT.md` 协助前端对接文件管理、后台任务、回收站和搜索接口。
2. 根据真实前端接入反馈，小步收敛不顺手的 API 模型、错误提示和状态字段。
3. 为 Docker 性能冒烟脚本补充耗时摘要或可选更大参数，但不扩展成复杂压测系统。
4. 继续维护错误码测试、部署文档和性能边界。

## 已推进

- 昂贵的目录总数统计已改为 `includeTotal=true` 可选。
- 目录总数按需返回已有 API 冒烟保护：默认分页响应不包含 `folderTotal/fileTotal`，显式 `includeTotal=true` 才返回总数。
- 目录分页上限已有 API 冒烟保护：请求 `limit` 会被 `WEB_FILE_BROWSER_MAX_DIR_PAGE_SIZE` 截断，`limit=0` 返回稳定 JSON 错误。
- 目录昂贵排序已有 API 冒烟保护：`detail=basic` 禁止按大小或修改时间排序，只有显式 `detail=full` 才执行需要条目元数据的排序。
- `/api/file/*` 元数据响应已改为复用同一次 JSON 序列化结果生成 ETag 和响应体；`Last-Modified` 直接从元数据结构计算，避免大目录响应双重序列化和 JSON 反解析。
- `/api/file/*` 元数据条件请求已支持 `If-None-Match` 和 `If-Modified-Since`，命中时返回 `304`；默认目录响应会复用目录自身 mtime 生成 `Last-Modified`，没有 `If-None-Match` 且 `If-Modified-Since` 命中时会跳过 JSON 序列化，减少重复传输且不额外扫描子项元数据。
- `detail=basic` 的元数据请求会在没有 `If-None-Match` 时用目标自身 mtime 做轻量 `If-Modified-Since` 预检查，命中可在占用目录扫描并发许可前直接返回 `304`。
- 按名称排序的目录分页读取已改为只保留当前页所需的排序窗口，仍完整扫描目录保证结果正确，但避免把页外条目全部留在内存中。
- 现有写入口已接入冲突策略，默认自动重命名；压缩和解压任务已复用同一策略；回收站恢复 API 冒烟已覆盖默认自动重命名、显式拒绝和显式覆盖；同步和异步文件覆盖替换已改为先保护旧目标，避免替换失败时先删除旧文件。
- 内容接口已返回 ETag，保存接口已通过 `If-Match` 防止旧内容覆盖外部修改。
- 普通 `HEAD /api/content/*` 和 `HEAD /api/download/*` 已改为只读取元数据，不打开文件句柄；Range HEAD 和非法 Range 已有 API 冒烟保护。
- 编辑器读取已切换到 `mode=edit`，后端会按编辑大小上限和文本/二进制判断保护在线编辑。
- 在线编辑白名单已完成基础实现：可通过环境变量配置扩展名或 MIME 分组，未配置时保持现有自动文本判断。
- 已增加单管理员密码修改 API，修改成功后旧会话失效，当前请求获得新会话；服务端会话会按 7 天有效期懒清理。
- 默认 CORS 已收紧为同源；跨域访问必须显式配置可信来源，且不支持 `*`。
- 已新增 Dockerfile、`.dockerignore`、Compose 示例、`env.example` 和 Linux Docker 部署文档。
- Docker 功能冒烟脚本 `scripts/docker-smoke.sh` 已在真实 Linux Docker 环境通过，使用 `.smoke/docker` 临时目录验证前端静态托管、登录、挂载、编辑保存、下载、上传、zip/tar.gz 压缩和解压、删除/恢复和指标接口。
- Docker 性能冒烟脚本 `scripts/docker-perf-smoke.sh` 已在真实 Linux Docker 环境按默认参数通过，使用 `.smoke/perf` 临时目录验证 1 万项目录分页、按需总数、大文件上传下载、Range、编辑保护、压缩解压、回收站恢复和指标接口。
- 后端 API 契约文档 `docs/API_CONTRACT.md` 已新增，后续前端对接应优先参考该文档和 `docs/API_ERRORS.md`。
- 启用搜索索引不再触发启动全量扫描；只有显式开启启动重建开关才扫描挂载目录。
- 已新增压缩和解压后台任务：支持 `tar.gz`、`tgz` 和 `zip`，读写按块推进，解压会拒绝路径穿越、符号链接、不支持的条目类型和超过深度上限的条目。
- 压缩和解压大文件 API 冒烟已覆盖 zip 和 tar.gz，验证超过单块缓冲区的文件会正确累计任务字节并完整解压。
- 主界面已补充压缩和解压最小入口，可从工具栏或右键菜单创建后台任务。
- 主界面已补充后台任务面板：可手动刷新、查看任务状态、进度、速度、错误数量，并可取消排队或运行中的任务。
- 后台任务历史已增加轻量内存清理：默认只保留最近 200 条已结束任务，排队和运行中的任务不参与历史清理；`POST /api/tasks/cleanup` 可主动清理已结束任务。
- 审计 JSONL 已增加按大小轮转和轮转文件数量保留：默认单文件 10 MiB、保留 8 个旧轮转文件；启动时只读取当前文件大小，后续通过进程内累计大小判断是否轮转，旧轮转清理只在新轮转或显式调用 `POST /api/audit/cleanup` 后触发。
- 回收站保留策略已支持请求触发：启动时只加载回收站索引，不默认扫描回收站内容；`POST /api/trash/cleanup` 可按天数和容量清理，`GET /api/trash` 会按间隔触发一次保留检查。
- 回收站普通文件记录已包含 `sizeBytes`，容量保留策略会优先使用索引中的大小，只有缺少大小的记录才按需访问磁盘。
- 回收站恢复已按当前挂载快照重新解析原虚拟路径，目标父目录不存在、挂载已移除或挂载只读时都会失败。
- 回收站恢复冲突策略已补充路由层覆盖：`reject` 会保留当前目标和回收站记录，`overwrite` 会替换目标文件并清理回收站记录。
- 后台任务取消响应已增强：压缩、解压和复制在更多文件系统边界检查取消状态，限速等待也会分片检查取消；移动任务在真实移动前会检查取消；复制任务会先写入同目录临时路径，取消或失败会清理临时输出，并在覆盖复制时保留旧目标直到完整复制成功；删除任务不再为了进度字节预扫目录。
- 后台复制任务已在用户选择的源路径和递归目录条目两层拒绝符号链接和不支持的特殊文件，避免跟随链接造成越界或循环。
- 移动和删除已拒绝直接操作符号链接，避免最终路径解析跟随链接后误移动或误删真实目标。
- 复制、压缩和解压已统一使用批量进度上报：小块读写按字节阈值或时间间隔合并更新任务状态，减少大文件操作中的跨线程锁竞争，成功收尾前刷新精确字节数，并保留低延迟取消检查。
- `/api/metrics` 已补充轻量内存指标快照：并发占用、任务状态分布、回收站条目数和索引状态；默认不为了指标递归扫描磁盘。
- `/api/metrics` 已有 API 冒烟保护：映射、会话、回收站、限流、任务和索引状态字段都来自轻量运行时快照。
- 单 IP 并发保护已补充 API 冒烟测试：挂起的上传请求会持续占用来源 IP 的并发许可，同 IP 后续受保护接口会返回 `429 TOO_MANY_REQUESTS`，释放后指标计数回落；默认模式下伪造不同 `X-Forwarded-For` 也不能绕过连接 IP 限流。
- 单 IP 并发保护会在请求和指标路径顺手清理空闲 IP 记录，避免长期运行后历史来源地址无限积累。
- `/api/ready` 已作为 Docker 就绪检查：验证管理员密码初始化和必要运行目录可用，不扫描挂载目录。
- `/api/ready` 文件系统探测已移到阻塞线程池，避免 Docker 健康检查在异步运行时里执行同步 I/O；成功和未初始化管理员密码场景已有 API 冒烟保护。
- API 错误码文档已新增：`docs/API_ERRORS.md` 记录统一错误响应、稳定 code、典型场景和前端处理建议；错误 code/status 映射已补充单元测试保护。
- 后台任务状态已补充当前处理条目 `currentPath`，只在已有处理循环中更新，任务结束后清空，不为了展示状态额外扫描磁盘。
- 搜索索引已接入应用内增量更新；批量复制和解压只刷新输出根路径，不为了索引递归扫描新目录。
- 挂载配置变化会按挂载点清理内存索引，避免旧路径残留；不会因此自动扫描新挂载目录。
- 搜索查询分页只克隆当前页结果，避免大索引查询时产生不必要的匹配结果副本。
- 搜索索引条目会缓存小写虚拟路径，查询直接复用缓存做大小写不敏感匹配，不为每次搜索重复生成每个条目的名称、路径和扩展名小写副本。
- 最近文件查询只维护请求数量范围内的候选窗口，并使用索引条目缓存的数值修改时间排序，不再为了返回少量结果对整份搜索索引引用做全量排序。
- 搜索索引重建可取消，但取消只在目录和条目边界生效，不为了强制中断单个文件系统调用引入复杂运行时。
- 搜索索引重复重建请求会在路由层返回 `409 CONFLICT`，避免已有重建仍在运行时继续接受新的重建请求。
- API 冒烟测试已补充只读挂载写入失败、路径越界拦截、跨挂载移动拒绝、后台移动任务取消和后台复制到只读挂载失败，确保常见安全边界在路由层返回稳定 JSON 错误或任务错误。
- 审计边界 API 冒烟测试已补充：登录、保存、上传、删除和回收站恢复会写入审计；目录列表、内容读取和下载不会为了低价值事件产生额外 JSONL 写入。
- 任务和审计清理 API 冒烟测试已补充：任务清理只删除已结束任务，审计清理只删除旧轮转文件。
